# 📱 Detox Campus: 대학생 건강 지표 관리 및 소셜 동기부여 앱

**Detox Campus**는 대학생들이 일상에서 쉽게 노출되는 유해 물질과 불규칙한 생활 습관을 기록·추적하고, 파티원 간의 실시간 상호 견제를 통해 건강한 캠퍼스 라이프를 만들어가는 헬스케어 안드로이드 애플리케이션입니다. (디자인 씽킹 프로젝트)

---

## 💡 프로젝트 개요 (Overview)
- **주요 목적:** 혼자서 제어하기 힘든 부정적 건강 지표(과다 카페인, 음주, 흡연, 스크린 타임 등)를 파티 기반의 소셜 기능을 통해 재미있고 지속 가능하게 관리 및 정화(Detox).
- **핵심 차별점:** 공신력 있는 기관의 기준에 맞춘 임계치 패널티 및 대사 반감기 로직 적용, 파티원 간의 실시간 '콕 찌르기' 푸시 알림을 통한 상호 견제 시스템.
- **개발 환경:** Android (Kotlin, Jetpack Compose), Node.js 푸시 서버, Firebase (Firestore, FCM)

---

## ✨ 핵심 기능 (Key Features)

### 1. 3대 유해 물질 직접 입력 및 실시간 로그
- **대상 항목:** 알코올, 카페인, 흡연
- **유연한 단위 설정:** (소주 1잔 당 알코올 6g)과 같은 가이드라인 기본값을 제공하며, 사용자가 직접 새로운 항목 이름, 섭취 단위(잔, 병, 개비 등), 함량을 커스텀하게 기입할 수 있습니다.
- **직관적인 UI 인터랙션:** 사용자는 슬라이더(Slider) 조작을 통해 간편하게 섭취량을 조절하거나, 직접 입력 칸에 정밀한 숫자를 기입하여 카운팅할 수 있습니다.
- **실시간 히스토리:** 데이터를 기입하는 즉시 하단 로그 섹션에 시간과 함께 기록이 누적됩니다.

### 2. 라이프 밸런스 지표 자동 연동
사용자의 입력 피로도를 줄이기 위해 디바이스 자체에서 측정되는 생활 패턴 데이터를 백그라운드에서 자동으로 수집합니다.
- **대상 항목:** 스크린타임, 활동 시간, 수면 시간
- **연동 기술:** **Google Health Connect API** 연동을 통해 사용자가 앱을 켜지 않아도 해당 지표들을 실시간으로 자동 산정 및 반영합니다.

### 3. 대시보드 및 건강 히트맵 (Health Heatmap)
- **기본 점수제:** 모든 사용자는 매일 **기본 100점**에서 시작하여 실시간 감점 수치가 차감됩니다.
- **비주얼 가이드:** 잔여 점수와 건강 상태의 심각도에 따라 색상 그라데이션이 변하는 **'건강 히트맵'** UI를 통해 유저에게 즉각적인 경각심을 줍니다.

### 4. 소셜 파티 및 양심 자극 시스템
- **방 생성 및 입장:** 소셜 파티 탭에서 고유한 초대 코드를 생성하여 파티를 개설하거나, 코드를 입력해 타인의 파티에 입장할 수 있습니다.
- **실시간 리더보드:** 파티원들의 현재 건강 총점을 기준으로 순위를 매겨 실시간으로 정렬하는 경쟁 시스템을 제공합니다.
- **상세 로그 모니터링:** 특정 파티원을 선택하면 해당 유저의 알코올, 카페인, 흡연 상세 로그와 구체적인 감점 내역을 열람할 수 있습니다.
- **콕 찌르기 (Nudge):** 파티원의 상태가 위험해 보이거나 실제 기록을 누락한 것으로 의심될 경우, 실시간 푸시 알림을 발송함으로써 양심적인 기록과 행동 절제를 자극합니다.

## 🧮 점수 산정 로직 (Scoring System Logic)

### 1. 임계치 기준 폭주 패널티 (Runaway Penalty)
직접 입력하는 3대 유해 물질은 공신력 있는 기관(WHO, 식품의약품안전처, 질병관리청 등)의 권고 기준에 의거하여 임계치가 설정됩니다.
- **임계치 이하 섭취 시:** 각 항목에 지정된 표준 가중치 $W$에 따라 감점됩니다.
- **임계치 초과 시 (폭주 상태):** 신체 안전 기준선을 넘는 순간부터는 폭주 패널티가 적용되어, 기존 가중치의 2배($2W$)만큼 감점 수치가 가속화됩니다.

### 2. 시간 경과에 따른 신체 잔류량 반감기 감쇄 로직
섭취 및 흡연으로 인한 감점은 체내에서 대사되거나 시간이 흐름에 따라 점차 회복되는 신체 메커니즘을 반영합니다. 감점 발생 시점을 기준으로 소요 시간($t$, 단위: 시간)에 따라 감점 수치가 지수적으로 감소합니다.

- **알코올 및 흡연 반감기 공식 (24시간 주기 반감):**
  $$D(t) = D_0 \times \left(\frac{1}{2}\right)^{\frac{t}{24}}$$

- **카페인 반감기 공식 (6시간 주기 반감):**
  $$D(t) = D_0 \times \left(\frac{1}{2}\right)^{\frac{t}{6}}$$

*(단, $D_0$는 최초 섭취 시 발생한 원본 감점 수치이며, $t$는 섭취 후 경과한 시간이다.)*

> **⚠️ 일주일 누적 초기화 규칙**
> 알코올, 카페인, 흡연 세 가지 직접 입력 항목은 섭취 시점으로부터 정확히 일주일($t \ge 168$)이 경과하는 순간 잔여 점수에 상관없이 무조건 0점으로 처리되어 완전 정화 상태로 리셋됩니다.

## 🏗 시스템 아키텍처 (Architecture)

본 프로젝트는 클라이언트가 알림을 직접 발송하지 않고, 데이터베이스 감시를 통해 서버가 중계하는 **신뢰성 높은 3-Tier 아키텍처**를 채택했습니다.

1. **Android Client:** 사용자가 콕 찌르기 실행 ➔ Firestore `Interactions` 컬렉션에 데이터(Log) 기록.
2. **Node.js Push Server:** 백그라운드(PM2)에서 Firestore를 실시간 감시(`onSnapshot`)하다가 새 데이터 감지.
3. **Firebase Cloud Messaging (FCM):** Node.js 서버가 수신자의 FCM 토큰을 식별하여 Google 서버를 통해 푸시 알림(Payload) 발송.

## 🛠 기술 스택 (Tech Stack)

### Frontend
- **Language:** Kotlin
- **UI Toolkit:** Jetpack Compose, Material3
- **Architecture:** MVVM (Model-View-ViewModel)
- **OS API:** Google Health Connect API (라이프 지표 자동 수집)

### Backend & Infrastructure
- **Server:** Node.js (PM2를 이용한 백그라운드 무중단 운영)
- **Database:** Firebase Firestore (NoSQL)
- **Push Notification:** Firebase Cloud Messaging (FCM HTTP v1 API)

## 🚀 설치 및 실행 방법 (Getting Started)

### 1. Android 클라이언트 빌드
1. 본 저장소를 Clone 합니다.
2. Android Studio에서 프로젝트를 엽니다.
3. `google-services.json` 파일을 `app/` 디렉토리에 추가합니다.
4. 기기 또는 에뮬레이터를 연결하고 `Run (Shift + F10)`을 실행합니다.

### 2. Node.js 푸시 서버 실행 (로컬/노트북 환경)
푸시 알림이 정상 작동하려면 백엔드 서버 가동이 필수적입니다.
```bash
# 1. 서버 디렉토리로 이동
cd push-server

# 2. 필수 라이브러리 설치
npm install firebase-admin

# 3. PM2 설치 (백그라운드 실행용)
npm install -g pm2

# 4. 서버 가동
pm2 start app.js --name "detox-campus-server"
